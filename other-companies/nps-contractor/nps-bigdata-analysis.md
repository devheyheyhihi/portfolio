# NPS 외주업체 빅데이터 분석 프로젝트

## 🏢 프로젝트 정보

**발주처**: 국민연금공단 (National Pension Service, NPS)  
**수행사**: 좋은정보기술 
**담당 역할**: 데이터 엔지니어 

## 📊 프로젝트 개요

국민연금공단 발주 빅데이터 분석 프로젝트에 외주업체 소속으로 참여하여, 전주시청과 협력하여 지역 농산물 및 유산균 데이터를 분석하는 업무를 수행했습니다. Python과 Apache Spark를 활용한 대용량 데이터 처리 및 분석 경험을 쌓았습니다.

## 🎯 프로젝트 목표

### 지역 데이터 분석을 통한 정책 지원
- **농산물 데이터 분석**: 전주 지역 농산물 생산, 유통, 소비 패턴 분석
- **유산균 데이터 분석**: 발효 식품 관련 데이터 수집 및 분석
- **정책 수립 지원**: 데이터 기반 의사결정을 위한 분석 자료 제공
- **지역 경제 활성화**: 농산물 산업 활성화를 위한 인사이트 도출

## 💼 주요 업무

### 1. 빅데이터 인프라 구축 및 운영

#### Apache Spark 기반 데이터 처리
- **분산 처리 환경 구축**: Spark Cluster 환경에서 대용량 데이터 처리
- **PySpark 활용**: Python과 Spark를 결합한 데이터 파이프라인 구축
- **성능 최적화**: 파티셔닝, 캐싱 등을 통한 처리 성능 개선
- **배치 처리**: 정기적인 데이터 수집 및 처리 자동화

#### 데이터 수집 및 전처리
- **다양한 데이터 소스**: 전주시청 데이터, 농산물 거래 데이터, 유산균 연구 데이터
- **데이터 정제**: 결측치 처리, 이상치 탐지 및 제거
- **데이터 변환**: 분석에 적합한 형태로 데이터 구조 변환
- **데이터 검증**: 데이터 품질 검증 및 무결성 확인

### 2. 농산물 데이터 분석

#### 생산 데이터 분석
- **생산량 추이 분석**: 시계열 데이터를 활용한 농산물 생산량 변화 분석
- **작물별 분석**: 주요 작물별 생산 패턴 및 특성 파악
- **계절별 분석**: 계절에 따른 생산량 변동 분석
- **지역별 비교**: 전주시와 타 지역 생산량 비교 분석

#### 유통 및 소비 패턴 분석
- **유통 경로 분석**: 생산자 → 도매 → 소매 → 소비자 경로 분석
- **가격 변동 분석**: 시기별 농산물 가격 변동 요인 분석
- **소비 트렌드**: 소비자 구매 패턴 및 선호도 분석
- **수급 예측**: 과거 데이터 기반 미래 수급 예측 모델

#### 상관관계 분석
- **기후 데이터 연계**: 기온, 강수량 등 기후 데이터와 생산량 상관관계 분석
- **경제 지표 연계**: 소득, 물가 등 경제 지표와 소비 패턴 상관관계
- **지역 특성**: 지역별 농업 환경과 생산성 관계 분석

### 3. 유산균 데이터 분석

#### 발효 식품 데이터 분석
- **유산균 종류별 분석**: 다양한 유산균 종류 및 특성 데이터 분석
- **발효 조건 분석**: 온도, 습도, 시간 등 발효 조건과 품질 관계 분석
- **품질 평가**: 유산균 수, pH, 산도 등 품질 지표 분석
- **최적 조건 도출**: 데이터 기반 최적 발효 조건 도출

#### 생산 공정 데이터 분석
- **공정 모니터링**: 실시간 생산 공정 데이터 수집 및 모니터링
- **불량률 분석**: 공정별 불량 발생 패턴 분석
- **효율성 개선**: 데이터 기반 공정 효율성 개선 방안 도출

#### 연구 데이터 분석
- **실험 데이터 분석**: 유산균 관련 실험 데이터 통계 분석
- **효능 분석**: 유산균 종류별 건강 효능 데이터 분석
- **문헌 데이터**: 관련 연구 논문 및 특허 데이터 분석

### 4. 전주시청 협업

#### 정기 미팅 및 보고
- **주간/월간 미팅**: 분석 진행 상황 보고 및 피드백 수렴
- **분석 결과 발표**: 시청 담당자 대상 분석 결과 프레젠테이션
- **정책 제안**: 데이터 분석 결과 기반 정책 개선안 제안

#### 데이터 협업
- **데이터 요청**: 필요한 데이터 항목 및 형식 협의
- **데이터 제공**: 시청에서 보유한 데이터 수령 및 검토
- **데이터 보안**: 개인정보 보호 및 데이터 보안 정책 준수

#### 요구사항 관리
- **분석 요구사항 수렴**: 시청에서 필요로 하는 분석 항목 파악
- **우선순위 조정**: 분석 항목 우선순위 협의 및 조정
- **결과물 검토**: 분석 결과물 시청 검토 및 피드백 반영

## 🎓 배운 점

### 빅데이터 실무 경험
Spark를 활용한 대용량 데이터 처리를 실제 프로젝트에 적용하면서, 분산 컴퓨팅의 원리와 실무적인 최적화 기법을 체득했습니다. 단순히 도구를 사용하는 것을 넘어, 데이터의 특성에 따라 적절한 처리 방법을 선택하고 성능을 최적화하는 능력을 키웠습니다.

### 공공 데이터 분석
공공기관에서 수집한 데이터를 다루면서, 데이터 품질 이슈, 개인정보 보호, 데이터 보안 등 공공 데이터 특유의 과제들을 경험했습니다. 또한 분석 결과가 실제 정책 수립에 활용될 수 있다는 점에서 데이터 분석의 사회적 가치를 실감했습니다.

### 도메인 지식의 중요성
농산물과 유산균이라는 전혀 다른 두 도메인의 데이터를 분석하면서, 데이터 분석 기술만큼이나 해당 도메인에 대한 이해가 중요하다는 것을 배웠습니다. 도메인 전문가들과의 소통을 통해 단순한 수치를 넘어 의미 있는 인사이트를 도출할 수 있었습니다.

### 협업의 중요성
외주업체로서 발주처(NPS), 협력기관(전주시청)과 함께 일하면서 다양한 이해관계자와의 효과적인 협업 방법을 배웠습니다. 기술적인 내용을 비기술자에게 전달하고, 요구사항을 정확히 이해하여 분석에 반영하는 커뮤니케이션 능력의 중요성을 깨달았습니다.

## 🔗 관련 프로젝트 연계

### 엑셀에서 빅데이터로
이전 KTL에서의 엑셀 기반 데이터 관리 경험이 빅데이터 분석의 기초가 되었습니다. 엑셀에서 수행하던 데이터 정제, 집계, 분석 작업들이 Python과 Spark를 통해 대규모로 확장되었고, 이를 통해 빅데이터 기술을 빠르게 습득할 수 있었습니다.

### 데이터 분석에서 개발로
빅데이터 분석 프로젝트를 통해 Python 프로그래밍 역량을 크게 향상시켰고, 이는 이후 웹 개발 및 블록체인 개발로 이어지는 기술적 기반이 되었습니다. 데이터 분석가로서의 경험은 개발자로서 데이터 중심의 사고방식을 갖추는 데 큰 도움이 되었습니다.
